# 프로젝트 요구사항

## 1. 개요
* 목적 : 확장 방향을 고려한 소규모 SNS 구현
* 기술 스택 : 바닐라 JS(프론트) + Express.js + MongoDB + Redis
  * MongoDB 는 로컬, Redis 는 로컬 WSL에 있는 것을 사용
* 프론트 스타일 : 깔끔하고 적당히 이쁘장하게, 구현이 쉬운 단순 레이아웃


## 2. 사용자 유형 및 권한
* 비로그인 방문자
  * 글 목록/상세 열람, 메인 타임라인 열람만 가능
* 일반 유저
  * 글·댓글 작성, 좋아요, 팔로우, 본인 콘텐츠 수정·삭제
* popular
  * 일반 유저의 모든 권한 + popular 식별 표시
  * 승격 조건 : 팔로워 10명 이상 && 관리자 수동 승격
* 관리자
  * 전체 콘텐츠·회원에 대한 관리 권한


## 3. 회원
* 가입
  * 이메일 + 비밀번호 + 닉네임
  * 이메일/닉네임 중복 불가
* 인증
  * JWT 로그인
  * Refresh Token 패턴(access 짧게 + refresh 길게), refresh 는 Redis 에 저장
  * 토큰 회전
* 프로필
  * 닉네임, 프로필 이미지 편집 가능
* 팔로우
  * 팔로우 / 언팔로우
* 탈퇴
  * 본인 요청에 의한 탈퇴
  * 관리자 제재에 의한 강제 탈퇴


## 4. 게시판
* 글
  * 제목, 내용 
  * 이미지 첨부 가능(3개까지 첨부 가능)
  * 조회수, 좋아요, 댓글
  * 작성·수정·삭제는 로그인 필요
  * 본인 글만 수정·삭제 가능, 관리자는 모두 가능
* 댓글
  * 작성·수정·삭제는 로그인 필요
  * 본인 댓글만 수정·삭제 가능, 관리자는 모두 가능
* 좋아요
  * 로그인 필요, 사용자당 글 1회
* 이미지 업로드
  * 로컬 디스크 저장(서버 내 업로드 디렉토리, 정적 서빙)
  * 허용 확장자 : jpg, png, webp
  * 용량 제한 : 1장 5MB 이내
* 프론트에서 보여질 형식
  * 글 목록 + 글 디테일 — 페이지네이션 , 글 목록에선 각각이 1줄로 보이게
  * 타임라인 — 메인화면에 조회수·작성날짜 기준 상위 10개


## 5. 관리자 페이지
* 회원 관리
  * 회원 목록 / 검색
  * 권한 변경, popular 수동 승격
  * 정지·강제 탈퇴
* 콘텐츠 관리
  * 게시글·댓글 삭제·숨김 처리
* 대시보드 통계
  * 가입자 수, 게시글 수, 일일 활성치 등 요약 지표


## 6. 비기능 요구사항
* 보안
  * 비밀번호 해싱 저장(bcrypt)
  * JWT 서명, 시크릿은 환경변수로 관리
  * 기본 XSS·인젝션 방어
* 에러 처리
  * 공통 에러 응답 포맷
* 로깅
  * 요청 로그 / 에러 로그
* 환경설정
  * `.env` 로 DB, Redis, JWT, 업로드 경로 등 분리
* 확장성
  * 세션 대신 JWT + Redis 사용
  * 이미지 경로는 추후 CDN·S3 등으로 교체 가능한 구조로 가정


## 7. 향후 확장 방향
* 단체 채팅 기능 추가
* Nginx, Docker 도입해 다중 WAS 구성
